# NeuralProphet

## 목표

Prometheus로 수집되는 트래픽 매트릭 데이터를 시계열 예측하여 트래픽이 많아질 구간을 예측하여 Node, Pod의 Scale out한다.

## 특징

- 5분마다 트래픽 매트릭 데이터를 활용해 추론
- 월드컵 경기 결과 사이트로 이벤트성 트래픽 발생
- 평소에는 트래픽이 적지만 **경기 시작 전, 경기 종료 직후, 승부차기, 골 발생**처럼 순간적으로 수십~수백 배 증가
- 예측 결과를 Karpenter가 Node Scale-out에 사용
- KEDA가 Pod Scale-out에 사용

즉, 정확한 장기 예측 보다는 **30분 ~ 1시간 앞의 트래픽을 최대한 빨리 예측**하여 미리 Node를 띄우는것이 목표

## 학습할 데이터 선정

NeuralProphet의 y값으로 활용할 데이터로 HTTP 요청 rate를 사용하는게 프로젝트에 적합

## NeuralProphet 모델 생성 옵션

옵션 선정 시 고려 사항

- 추론을 5분마다 수행하고, 모델 학습은 하루에 1번만 수행.
- 이벤트성 트래픽.
- 최근 데이터는 추론 시접에 입력(n_lags)으로 반영되므로 최근 패턴을 충분히 볼 수 있어야함.
- 학습은 하루에 한번만 이루어지므로 하루동안 발생하는 반복 패턴(경기 시작, 하프타임, 경기 종료)을 일반화해서 학습해야 함.

위 고려사항을 보면 자기회귀(과거를 통한 미래 예측)보다는 계절성(Seasonality)와 이벤트 정보를 적극 활용하는것이 좋아 보임.
모델을 자주 학습시키는 것보다 안정적인 일반화 성능이 중요하다고 봄.

##### 왜 이런 옵션을 선택했는지

- Trend: 장기적인 사용자 증가나 감소를 학습
- Seasonality: 경기 시간, 요일 등 반복되는 패턴을 학습
- Auto Regression: 최근 3시간의 트래픽 흐름을 반영하여 단기 예측 수행
- Forecast: 1시간 후까지 예측하여 Karpenter가 노드를 미리 생성할 시간을 확보
- Training: 하루 1회 학습으로 모델을 최신 상태로 유지
- Data: 결측치 보완과 표준화를 통해 안정적인 학습 수행

## 중요

해당 프로젝트에서 옵션보다 중요한 요소가 있다.
경기 일정을 모델에게 알려주는 것이 예측 정확도에 큰 영향을 미친다.

예를 들어 18:00 한일전 이라는 빅매치가 성사될 경우 기존 트래픽만 가지고 예측을 할 경우 예상치 못한 트래픽이 발생할 수 있다.

따라서 추가해야할 기능으로

`model.add_events(["match_start"])`

또는

`model.add_events(["match_start", "half_time", "second_half", "match_end"])`

처럼 경기와 관련된 정보를 모델에게 알려준다.

경기정보를 전달할 경우
경기시작 -> 트래픽 증가 -> 하프타임 -> 잠깐 감소 -> 후반 시작 -> 다시 증가 -> 종료 -> 감소
와 같은 싸이클로 Scale-out을 할 수 있다.
